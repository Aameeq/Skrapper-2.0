Getting Started

What is Skraper Web?

Skraper Web is a powerful web application that leverages the Skraper library to extract structured data from 18+ social media platforms without requiring authentication or API keys. It provides JSON-formatted data that can be used for analysis, research, or integration into other applications.

Key Features

  • ‚Ä¢ Support for 18+ social media platforms
  • ‚Ä¢ No authentication or API keys required
  • ‚Ä¢ Multiple output formats (JSON, CSV, XML, YAML)
  • ‚Ä¢ Real-time processing with progress tracking
  • ‚Ä¢ Interactive data visualization and analysis

Quick Start

1. Enter a social media URL in the input field

2. Configure your scraping preferences

3. Click "Start Scraping" to begin extraction

4. View results in the interactive dashboard

5. Export data in your preferred format

API Reference

Response Format

All responses follow a consistent JSON structure:

{
  "metadata": {
    "url": "https://instagram.com/username",
    "platform": "instagram",
    "scraped_at": "2024-01-15T10:30:00Z",
    "content_type": "posts",
    "output_format": "json",
    "limit": 50
  },
  "data": [
    {
      "id": "post_1",
      "username": "sample_user",
      "content": "Post content here",
      "timestamp": "2024-01-14T15:30:00Z",
      "likes": 245,
      "comments": 12,
      "shares": 3,
      "media_url": "https://example.com/media.jpg",
      "caption": "Post caption",
      "hashtags": ["#tag1", "#tag2"],
      "mentions": []
    }
  ],
  "statistics": {
    "total_posts": 20,
    "media_items": 35,
    "engagement_metrics": {
      "total_likes": 15420,
      "total_comments": 892,
      "total_shares": 234
    }
  }
}

Data Fields

Post Object

  • id: Unique post identifier
  • username: Post author
  • content: Post text content
  • timestamp: ISO 8601 date
  • likes: Like count
  • comments: Comment count
  • shares: Share count
  • media_url: Media file URL
  • hashtags: Array of hashtags
  • mentions: Array of mentions

Statistics Object

  • total_posts: Number of posts
  • media_items: Media files count
  • engagement_metrics: Engagement data
  • total_likes: Sum of all likes
  • total_comments: Sum of comments
  • total_shares: Sum of shares

Data Analysis

Analyze engagement patterns:

# Sample Analysis with Python
import json
import matplotlib.pyplot as plt

# Load scraped data
with open('scraped_data.json', 'r') as f:
    data = json.load(f)

# Extract engagement metrics
posts = data['data']
likes = [post['likes'] for post in posts]
dates = [post['timestamp'][:10] for post in posts]

# Create engagement chart
plt.figure(figsize=(12, 6))
plt.plot(dates, likes, marker='o')
plt.title('Engagement Over Time')
plt.xlabel('Date')
plt.ylabel('Likes')
plt.xticks(rotation=45)
plt.tight_layout()
plt.show()

Export to Database

Store results in a database:

# MongoDB Example
from pymongo import MongoClient
import json

# Connect to MongoDB
client = MongoClient('mongodb://localhost:27017/')
db = client['social_media']
collection = db['scraped_posts']

# Load and insert data
with open('scraped_data.json', 'r') as f:
    data = json.load(f)
    collection.insert_many(data['data'])

print(f"Inserted {len(data['data'])} posts")

Frequently Asked Questions

Is this service free to use?

Yes, Skraper Web is currently free for basic usage. However, please be mindful of rate limits and use the service responsibly.

Do I need API keys?

No, Skraper Web works without any API keys or authentication. It uses public data scraping techniques to extract information.

How accurate is the data?

The data accuracy depends on the platform and current website structure. We strive for 99%+ accuracy, but platform changes can temporarily affect results.

Can I scrape private profiles?

No, Skraper Web only accesses publicly available content. Private profiles, restricted content, and content requiring login cannot be scraped.

What if scraping fails?

If scraping fails, try again later. The platform may have made changes, or there might be temporary connectivity issues. Check our status page for known issues.

How often can I scrape?

There's no strict limit, but we recommend reasonable intervals between requests to avoid being blocked by platforms. Consider implementing delays in your workflow.
